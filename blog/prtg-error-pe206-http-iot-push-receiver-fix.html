PRTG Error PE206 Oplossen: HTTP IoT Push Receiver Fix

Door Thomas Faddegon 26 september 2025 PRTG Troubleshooting 4 min leestijd

Krijg je deze error bij het opzetten van een HTTP IoT Push Data Advanced sensor in PRTG?

PRTG could not start the HTTP push receiver.
error:80000002:system library::No such file or directory (code: PE206)

Hier is de snelle oplossing voor hosted PRTG instances met remote probes.

Wat is Error PE206?

PE206 betekent dat PRTG de SSL certificaten niet kan vinden in de \cert directory van je remote probe. Bij hosted instances kun je deze niet kopiÃ«ren van de core server.

Oplossing: Genereer Eigen Certificaten

PRTG documentatie zegt: "this certificate does not have to match the certificate that you use on the PRTG core server" - dus we maken gewoon onze eigen certificaten!

Tip: Heb je een interne Certificate Authority (CA) in je Windows domein? Dan kun je beter een certificaat aanvragen via de CA in plaats van een self-signed certificaat te maken. Dit voorkomt browser waarschuwingen en geeft betere beveiliging.

Stap 1: Certificaten Maken

# Maak werkdirectory
mkdir prtg-certs && cd prtg-certs

# Private key genereren
openssl genrsa -out prtg.key 2048

# Self-signed certificaat maken (365 dagen geldig)
openssl req -new -x509 -key prtg.key -out prtg.crt -days 365 \
-subj "/C=NL/O=PRTG/CN=prtg-remote-probe"

# Root certificaat maken
cp prtg.crt root.pem

Stap 2: Installeren op Remote Probe

  1. Stop PRTG Remote Probe Service (via PRTG Administration Tool)
  2. Kopieer bestanden naar: 
    C:\Program Files (x86)\PRTG Network Monitor\cert\prtg.crt
C:\Program Files (x86)\PRTG Network Monitor\cert\prtg.key
C:\Program Files (x86)\PRTG Network Monitor\cert\root.pem
  3. Start service opnieuw

Stap 3: Testen

HTTP IoT Push Data Advanced sensor toevoegen - PE206 error zou verdwenen moeten zijn!

Controleer of de TCP poort actief is op de probe server:

netstat -an | findstr 5051

Je zou een regel moeten zien zoals: TCP 0.0.0.0:5051 0.0.0.0:0 LISTENING

Test daarna met: https://<probe-ip>:5051/<token>?content=<XML_data>

Samenvatting

  • Probleem: PE206 = ontbrekende SSL certificaten
  • Oplossing: Eigen certificaten genereren met OpenSSL
  • Tijd: 5 minuten
  • Resultaat: Werkende IoT HTTP listener op poort 5051

Error opgelost!
